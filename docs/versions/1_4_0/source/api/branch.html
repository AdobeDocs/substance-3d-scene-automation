substance3d.api.scene.branch

create(parent, identifier, comment='')

Create a SceneBranch context object. A scene branch is a way to edit and modify the current scene in a defined context without impacting it like a session layer. Enter a scene branch with the keyword with, and all the instructions will be applied on this specific branch layer on top of the current scene, exit the indentation block, the branch’s layer is not applied anymore and all the next instructions will be applied on the current scene’s layer.

Example:

scn = scene.new()
scene.add_layer(scn, 'shoes.fbx')

# create a scene branch
sb_green = scene.branch.create(scn, 'green')
with sb_green:
  # in this block all the modification will be done only for this scene branch
  mat = material.create_solid_color(scn, 'green', (0.0, 1.0, 0.5))
  material.assign_to_objects(mat, scn.search(type='mesh'))

# back to the current scene, all the modifications done into the 'green' scene branch are not applied here
# but they still exist in the 'green' scene branch

More into the user documentation.

Parameters:

  • parent (StageHandler) – the scene from where the branch is applied

  • identifier (str) – the branch’s identifier

  • comment (str) – add a comment for this branch, if the branch is save as file the comment will show-up as stage metadata

Return type:

SceneBranch

save(parent, identifier, filename, apply=False)

Save an existing scene branch by giving a scene and an identifier

Parameters:

  • parent (StageHandler) – the scene where the branch is applied

  • identifier (str) – the branch’s identifier

  • filename (str | pathlib.Path) – a correct filename, supported extension usd, usdc, usda

  • apply (bool) – if True the saved file will be added to the current scene layer stack
