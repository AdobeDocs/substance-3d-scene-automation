Save the given scene into a valid file if filename is given the scene will be written here otherwise the value of scene.path will be used.

  • filename (str | pathlib.Path | None) – a valid file path

  • flat (bool) – if flat is True, the scene will be saved with all references or layers into the same file, There will be no more references to other files

add_layer(parent, filename, **kwargs)
The content of the file will be added as a layer onto the given scene.

Add content by layer is commonly used to layering content by tasks or departments, like lighting, cameras, animation… on top of the existing scene content. A bit like layers in photoshop, Each layer can have an influence on those below. To add an asset into a scene it’s more convenient to use a reference with scene.add_reference()

Parameters:

  • parent (StageHandler) – The scene on which the layer will be added

  • filename (str | pathlib.Path) – A valid file path, supported extensions: .ALLOWED_EXTENSIONS

  • kwargs – Give plugin specific arguments, call utility.get_plugins_sdfmetadata() to get more info

Return type:

StageHandler

add_reference(parent, filename, default_object=None, scale_to_scene_unit=False, adjust_orientation=True, **kwargs)
Reference an object and all its descendants under the given object. By default the object will be selected

accordingly to the file’s default object (defaultPrim) value. Add a reference is commonly used to add an asset to the given scene. As opposed to a layer, a reference allows to add the same asset multiple times by referencing it under different objects.

Parameters:

  • parent (PrimHandler) – The object on which the default file referenced object and its descendants will be added.

  • filename (str | pathlib.Path) – A valid file path, supported extensions: .ALLOWED_EXTENSIONS

  • default_object (str | None) – Give a file referenced object other than the default one (it must be at the root level)

  • scale_to_scene_unit (bool) – if True the referenced layer will be scaled to fit the parent scene unit

  • adjust_orientation (bool) – if True the referenced layer will be oriented accordingly to the scene up axis

  • kwargs – Give plugin specific arguments, call utility.get_plugins_sdfmetadata() to get more info

Return type:

PrimHandler

new_from_object(parents, filename=None, preserve_scene_metadata=True, flatten_parent_scene=False)

Create a new scene in memory with only the selected object(s) and its children

Parameters:

  • parents (list[substance3d.api.core.prim_handler.PrimHandler] | substance3d.api.core.prim_handler.PrimHandler) – the parent object(s) to “copy” into this new scene

  • filename (str | pathlib.Path | None) –

    a valid file name used as identifier and as default value if the scene is saved, extension must be ALLOWED_EXTENSIONS

    Warning

    the object’s asset dependency paths will not be changed therefore give a filename that will not break dependencies.

  • preserve_scene_metadata (bool) – if True the parent object’s scene metadata will be also copied into this new scene if several parent objects are given only the first object metadata will be copied

  • flatten_parent_scene (bool) – if True the selected object will copy from a flatten scene. Can be convenient when selected objects are in variants or with opinions from different targets

Returns:

the new scene created in memory

Return type:
